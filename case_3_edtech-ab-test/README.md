# A/B-тест персонализированной карусели курсов (EdTech)

## Описание проекта

Проект посвящён проектированию A/B-эксперимента для оценки влияния персонализированной карусели рекомендаций курсов «Вам понравится» на поведение пользователей образовательной платформы.

Цель проекта — продемонстрировать полный аналитический цикл **от анализа сырых логов пользовательских событий до обоснованного дизайна эксперимента**, включая выбор метрик, единицы рандомизации и оценку чувствительности теста.  
Фактические результаты A/B-теста в рамках проекта не анализируются.

Проект ориентирован на демонстрацию продуктового и экспериментального мышления аналитика данных.

## 1. Exploratory analysis и анализ данных  
**`1_exploratory_analysis.ipynb`**

В первом ноутбуке выполняется анализ исходных логов пользовательских событий и оценка качества данных.

Ключевые этапы анализа:
- описание структуры данных и событий;
- оценка плотности и разреженности пользовательских взаимодействий;
- проверка корректности логирования и логической целостности пользовательской воронки;
- анализ альтернативных пользовательских путей;
- формирование пользовательских воронок на разных уровнях агрегации;
- выбор основной единицы анализа (user-level);
- расчёт и описательная статистика продуктовых метрик.

Результаты анализа используются для обоснования ограничений данных и выбора корректного дизайна эксперимента.

---

## 2. Проектирование A/B-эксперимента  
**`2_experiment_design.ipynb`**

Во втором ноутбуке выполняется проектирование A/B-теста на основе выводов exploratory analysis.

Рассматриваются:
- формулировка продуктовой гипотезы;
- выбор ключевых, вспомогательных и охраняющих (guardrail) метрик;
- определение единицы рандомизации и уровня агрегации;
- описание дизайна эксперимента и основных допущений;
- анализ ограничений исторических данных;
- оценка минимального детектируемого эффекта (MDE) и требований к объёму выборки;
- использование прокси-метрик и рекомендация проведения A/A-теста перед запуском.

Ввиду отсутствия наблюдаемых конверсий в исторических данных расчёты MDE выполняются с использованием гипотетического baseline-значения и служат исключительно для иллюстрации чувствительности эксперимента, а не для прогнозирования эффекта.

---

## Основные выводы проекта

- Логи пользовательских событий характеризуются высокой разреженностью и наличием альтернативных пользовательских путей.
- Основной единицей анализа и рандомизации выбран пользователь (user-level), что обеспечивает корректность статистических выводов.
- При текущем объёме данных эксперимент способен обнаружить только достаточно крупные эффекты по ключевой метрике.
- Для повышения чувствительности эксперимента рекомендуется многоэтапный подход с использованием более чувствительных поведенческих метрик на ранних этапах.
